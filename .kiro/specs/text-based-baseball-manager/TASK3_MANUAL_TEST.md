# タスク3 マニュアルテスト手順

## テスト目的
打席判定エンジン（タスク3）が正しく動作することを確認する

## 前提条件
- 開発サーバーが起動していること（`npm run dev`）
- ブラウザで http://localhost:5175 にアクセス可能

## テストケース

### TC1: 基本的な打席判定フロー
**目的**: 三振/四球/ヒット/アウトの判定が正しく動作するか

**手順**:
1. ブラウザで http://localhost:5175 にアクセス
2. 「試合開始」ボタンをクリック
3. チーム選択画面で適当なチームを選択
4. 打順編集画面で「試合開始」をクリック
5. 1回表の攻撃が始まる
6. 攻撃指示メニューで「通常打撃」を選択

**期待結果**:
- 指示実行メッセージが表示される
- 1-2秒後に打席結果が表示される
- 以下のいずれかが表示される:
  - 「〇〇は三振しました。」
  - 「〇〇はフォアボールで出塁しました。」
  - 「〇〇が打ちました！」+ 打球の説明（例：「センター方向への強いゴロ」）
- プレイログに結果が記録される

### TC2: 打球の種類と方向の確認
**目的**: 打球の種類・方向・強さが適切に表示されるか

**手順**:
1. TC1と同様に試合を開始
2. 複数回「通常打撃」を選択して打席を繰り返す
3. プレイログに表示される打球の説明を確認

**期待結果**:
- 打球の種類: ゴロ/フライ/ライナー
- 打球の方向: 左方向/レフト前/センター方向/ライト前/右方向
- 打球の強さ: 弱い/中程度/強い/非常に強い
- 日本語で自然な説明が表示される

### TC3: ヒットと走者の進塁
**目的**: ヒットが出た場合、走者が正しく進塁するか

**手順**:
1. 試合を開始
2. 「通常打撃」を選択してヒットが出るまで繰り返す
3. ヒットが出たら、塁上の走者表示を確認
4. 次の打者でヒットを出し、走者の進塁を確認

**期待結果**:
- 単打: 打者が一塁に出塁、既存走者が1塁進む
- 二塁打: 打者が二塁に出塁、既存走者が2塁進む
- 三塁打: 打者が三塁に出塁、既存走者が本塁へ
- 本塁打: 全走者が本塁へ、得点が加算される
- 塁上の走者表示（◇/●）が更新される

### TC4: 得点の加算
**目的**: 得点が正しく加算されるか

**手順**:
1. 試合を開始
2. ヒットや本塁打を出して得点を重ねる
3. スコアボードを確認

**期待結果**:
- 得点が正しく加算される
- プレイログに「〇〇が1点を獲得！」などのメッセージが表示される
- スコアボードの数字が更新される

### TC5: 四球による進塁と押し出し
**目的**: 四球で走者が正しく進塁するか

**手順**:
1. 試合を開始
2. ヒットを連続で出して満塁にする
3. 四球が出るまで「通常打撃」を繰り返す

**期待結果**:
- 四球で打者が一塁に出塁
- 既存の走者が押し出される
- 満塁時の四球で1点が加算される（押し出し）

### TC6: アウトカウントと攻守交代
**目的**: アウトが3つで攻守交代するか

**手順**:
1. 試合を開始
2. アウトが出るまで「通常打撃」を繰り返す
3. 3アウトになるまで繰り返す

**期待結果**:
- アウトカウントが1→2→3と増える
- 3アウトで「チェンジ！」と表示される
- 次のイニングに進む（1回表→1回裏）

### TC7: 長打の発生
**目的**: 二塁打・三塁打・本塁打が適切に判定されるか

**手順**:
1. 試合を開始
2. 複数回試合を繰り返して長打が出るまで観察
3. 長打が出たときの走者の進塁を確認

**期待結果**:
- 強い打球で長打が出やすい
- 二塁打/三塁打/本塁打それぞれの判定が正しく動作
- プレイログに「〇〇の二塁打！」「〇〇のホームラン！」などが表示される

### TC8: 投球数の増加と疲労
**目的**: 投球数が増えると投手の能力が低下するか

**手順**:
1. 試合を開始
2. 複数イニング進行させる
3. 後半のイニングでヒットや四球が増えるか観察

**期待結果**:
- 投球数が増えるにつれて、四球やヒットが出やすくなる
- （目視確認のため、統計的な傾向を確認）

## 補足

- 打席判定は確率的なため、同じ操作でも結果が変わります
- 三振や四球が出にくい場合は、複数回試行してください
- 長打は低確率なので、発生するまで時間がかかる場合があります
